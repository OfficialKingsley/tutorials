Express JS

This is a web framework for node js. Previously, we used http and file system module to set up a web server. With express, it is easier and it does even more

Express is a fast, un-opinionated and minimalist web framework for node JS
It's not a high level framework like laravel or django it's very basic at its core. You have full control to the
It is backend/ server-side framework. IT's not a client-side framework

  • Makes building web applications with node js much easier
  • Used for both server rendered apps as well as API/Micro-servers
  • Extremely light, fast and free
  • Full control of request and response
  • Uses JS
  • By far the most popular
Basic Syntax 
const express = require("express")

//init express
const app = express();

//create your endpoints/route handlers

app.get('/', function(req, res) {
  res.send("Hello world");
});

//listen on a port
app.listen(3000, ()=>{console.log("Server started on port 3000")} )

Route Parameters

const express = require("express"); //This is a function
const app = express(); This function will return an object

app.get("/", (req, res)=> {
  res.send("Hello world")
})

app.get("/example", (req, res)=> {
  res.send("This is the example route")
})
app.get("/example/:name/:age", (req, res)=> {
  The params can be gotten from
  req.params

  This will return an object
  {name: "my name passed in the url", age: "age_passed_in_the_url"}
  res.send("This is the example route")
})
app.listen(3000)

Query strings

const express = require("express"); //This is a function
const app = express(); This function will return an object

app.get("/", (req, res)=> {
  res.send("Hello world")
})

app.get("/example", (req, res)=> {
  res.send("This is the example route")
})
app.get("/example/:name/:age", (req, res)=> {
  The params can be gotten from
  req.query
  //This will return an object
  //In the url, if we now type
  
  localhost:3000/hello_world?tutorial=my_tutorial,
  
  //we will see that if we console.log the req.query now, we will get
  {tutorial: "my_tutorial"}
  If you want to add another query, you should add and and sign &
  &size=100kb
  res.send("This is the example route")
})
app.listen(3000)

If the data from the url is compulsory, we use the route parameters

If we want to use something optional, we use query string
