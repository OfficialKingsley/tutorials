CSS grid

To create a grid we set the display of the element (or container). E.g.

.container {
display: grid;
}

You can specify the number of columns by defining the grid-template-colum.
The number of values you specify represent the number of columns you want while the values represent the actual size of the columns.

.container {
display: grid;
grid-template-columns: 100px 300px
}

You can also specify the number of rows using the grid-template-rows property.

You can also set the size of the row or column to auto. This value makes the row/column take up all the space that is left.

Example .container {
display: grid;
grid-template-columns: 100px 300px auto
}

Now we are going to have three columns. The first two columns have non-responsive values while the third has a responsive value and it takes up any remaining space after the first two have filled their positions.

Another value that can be used in grids is the fraction unit (represented as "fr").
What this unit does is that it divides the remaining space into the number of specified fractions. For example, if there are 3 columns with each having a value of 1fr, this will divide the screen into three equal parts with each column taking one fraction. This value is very helpful as it is responsive.

Also, if you make let's say the second one 2fr, the screen will be divided into four as the total sum of the fractions is 4fr. The second one is however going to take 2fr i.e. it is going to be bigger that the other two columns

The repeat function

This is used if you want to create rows or columns with the same size.
This function takes in two parameters which are the number of columns/rows that you want to apply it to and the second parameter is the size of each

Example .container {
display: grid;
grid-template-columns: repeat(3, 1fr)
}

Grid template

This is the short hand for the grid template columns and grid-template-rows

grid-template: grid-template-rows / grid-template-columns

Grid-row-gap

This sets a gap between rows

Grid-column-gap

This sets a defined gap between the columns

Grid-column-start and end

The grid-column-start is a property that is given to the items with the display of grid. It is not defined in the container. What this does is that it shows what line you want that column to start from. This also depends on the number of columns.
For example, if we have two columns defined in the container, we have three lines. The first line is the line or space before the start of the column. The second column line is the space between the two columns and the third column line ids the line the line after the second column

grid-column

This is the shorthand for the grid-column-start and end

grid-column: 1 / 3

If you want it to span across till the end, you use -1 as the line end

grid-column: 1 / -1

You can also use the span keyword. This now specifies the number of columns you want to span across. Note that this doesn't use column lines. It uses the actual columns

grid-column: 1 / span 2;

Note that when using the grid make sure they don't cut across each other

Grid-template-areas

This is put in the container.
It is like a visual representation of how you want the grid to look like. You are like naming the defined rows and columns.

.container {
display: grid;
grid-gap: 3px; grid-template-columns: repeat(12, 1fr);
grid-template-rows: 48px auto 48px;
grid-template-areas:
"h h h h h h h h h h h h"
"m c c c c c c c c c c c"
"f f f f f f f f f f f f"

Each pair of quotes represents a row and each letter in the row represents a cell

Grid-area

This property is given to the items and not the container. Here, the item is assigned a value. This value is a name of a cell defined in the grid-template-areas. So this item then takes up the space given to the cell

.container {
display: grid;
grid-gap: 3px; grid-template-columns: repeat(12, 1fr);
grid-template-rows: 48px auto 48px;
grid-template-areas:
"h h h h h h h h h h h h"
"m c c c c c c c c c c c"
"f f f f f f f f f f f f";
}
.header {
grid-area: h;
}
.main {
grid-area: m;
}
.content {
grid-area: c;
}
.footer {
grid-area: f;
}

You can also use dots(.) in css grid-template-areas to create empty spaces

auto-fit

this states that the grid should create the number of columns or rows based on the size of the screen of view width. However there is a downside to this as if the size of the space is not equal to the size of the column, then there will be extra space left which doesn't look nice. Also if you try repeat(autofit, 1fr), this will just create one column because autofit tries to fill up the grid with as many columns as possible until there's not enough space. But when you use 1fr both the width of the container and the column are the same

You can fix this using the minmax function

repeat (auto-fit, minmax(100px, 1fr))

The columns will always fit the entire grid irrespective of whether or not the container adds up to 100px. If the container has more space it distributes it across all the columns. This feature has more preference for the minimum than the maximum

Named lines

When defining your grid template column or grid template rows, you can give names to the column lines. his can be done using the square brackets

.container {
display: grid;
grid-gap: 3px; grid-template-columns: [main-start] 1fr [content-start] 5fr [content-end main-end]; <!--Notice that I used two names-->
grid-template-rows: [main-start]48px [content-start] auto [content-end] 48px [main-end];
.header {
grid-column: main-start/ main-end;
}
.menu {
grid-area: ;
}
.content {
grid-area: c;
}
.footer {
grid-area: f;
}

SInce the content part forms a box, and also because of the syntax we used for the naming, we can do

.container {
display: grid;
grid-gap: 3px; grid-template-columns: [main-start] 1fr [content-start] 5fr [content-end main-end]; <!--Notice that I used two names-->
grid-template-rows: [main-start]48px [content-start] auto [content-end] 48px [main-end];
.header {
grid-column: main; //!-start to -end
}
.menu {
grid-area: main-start / content-start;
}

}
.footer {
grid-column: main;
}

Justify Content and Align Content

Justify-content: controls the content along the x-axis. Values are:

  • start: Places all the content in the start of the grid
  • end
  • center

Justify Items and Align-items

Justify items is used to set the justification of the content. It is by default set to stretch so it can fill its container but it can also be set to start which places them in the center of their container.
