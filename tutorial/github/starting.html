First we open our terminal and we navigate to our desktop
We make a directory called story mkdir Story
Then you cd into Story
make a file called chapter1.txt
open the file
Next write something in it 

git init //This will create a local git repository to track our changes

It will create a file called .git (a hidden file that tracks your file)

In order to start tracking the changes of your file, you have to add the file to what's called a staging area and that is basically an intermediate place where you can choose the files in the working directory that you want to commit

To see what is currently in your staging area, you can type the command git status 

git status

Un-tracked files will be shown in red and this is something something in your working area but not in your staging area
In order to add it to the staging area, we have to type the command git add 

git add chapter1.txt


git status will show that chapter1.txt has been added

NOw to commit it, you type this

git commit -m "Git message"

You should always use present tense for your git messages

You can see the commits that you have made by typing the git log command

git log

now we can create more files and make changes to these files

git status

git add .

//This means add everything in this working directory

git commit -m "We Complete Chapter 2 and three"
//This commits your changes to your local repo

git log
Now we have two different commits 
//and each commit has a unique hash

If you make a mistake in your work

git status

git diff chapter3.txt
This shows the difference between the files

git checkout chapter3.txt
This rolls back chapter3.txt to the last state stored in the local repository

If you would like to add a description to your commit, you can type the code below

git commit -m "The commit message" -m "The decription"

The first -m flag is for the commit message and is very compulsory while the second is for the description

You can actually simply just type the code below to add (or track) the code and then commit it. However, it is advised that you actually add and then commit 

git commit -a -m "Commit message"
