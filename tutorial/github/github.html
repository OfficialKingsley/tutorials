Connecting your local computer to your github account

To do this, we'll need what is known as SSH keys.
We have to first start by typing in a command in the following way:
ssh-keygen -t type-of-encryption -b strength-of-encryption -C "github-email-address@email.com" 

ssh-keygen -t rsa -b 4096 -C "example@email.com"

After this an ssh-key will be generated.
The default file is in a file (C:/Users/user-name/.ssh/id_rsa). However you can change the file where it would be saved
Next, you pass in a passphrase and repeat the passphrase
In that folder you will have two keys which are id_rsa and id_rsa.pub
The one that you are going to upload is the one with the .pub and it is your public key

Create a repository on github
Give it a name and a description
You can set your repo type to either private or public

We can push our local repository to our github repo
Make sure you copy the link to your github repo 

git remote add origin url_of_your_repo
//This says that our remote repo is called origin

git push -u origin master
This pushes your local repository
//to origin (remote repo) to the master branch

The reason the -u flag is added is so that the you just type git push when you're in the master branch and then it is added to the

It should also be noted that if changes are made on github and you would like to get those changes from github, you can simply type the command below 

//If no upstream is set
git pull origin master

//If upstream already set,
git pull
