React Context API

React context helps us with an easy and clean way to share state without passing props down even to the components that do not need the props. This thereby makes react cleaner and easier
The components that don't use the props act as carrier components for the props
Using react with hooks the context api helps us to get something relative to redux.
You can share state within a component tree.

Examples and Usage

To use the context api, we have to first create a context then we make this context available to the component tree where it might be needed
To make it available or (known as providing it) using a context provider which is a react tag. We wrap the tree where we will need this data in the context provider element and then any element in the tree can access the context state.

When to use the context

It is advised to use the context api when the state to be shared can be considered as global or will likely be used by a lot of components. For example, a theme, a current authenticated user, preferred language.

Consuming a context

This simply means accessing the data in the context
