React Links

What we normally do is that we create links with the anchor tag. This works fine but it sends a get request to the server and the server always returns the index.html and then it loads the component and it makes the page to reload.
The disadvantage of this method is that it sends requests to the server and reloads the page which we do not want in a single page application.
Also, when the page is refreshed, we lose our state of our application at that point to the default state

To prevent this, we use the Link tags from the react router dom

      What we normally do is that we create links with the anchor tag. This
      works fine but it sends a get request to the server and the server always
      returns the index.html and then it loads the component and it makes the
      page to reload.

      Also, when the page is refreshed, we lose our state of our application at
      that point to the default state
    


    

      To prevent this, we use the Link tags from the react router dom. We should
      note that the Link tag does not have an href attribute but we have a to
      attribute and both of them perform the same action
    


    
When the link tag is rendered in the dom, it outputs as an anchor tag

NavLinks

    

      These are very similar to the Link tags but the difference is that
      NavLinks provide the active class to the active link. With this, we can
      style our links with the active class and any link that we click will then
      go to the route and then it gets styled as the active class will be passed
      to it
    

    

      If you want to change the className of the active link, you can specify
      this by adding an attribute called the activeClassName attribute. Note
      that this attribute has to be added to all the NavLinks in the specified
      navbar
    


    
Sub links or sub routes

    

      We can create routes in routes. To do this, we use the hook called the
      useRouteMatch hook from the react-router-dom
    

    

      The useRouteMatch hook gets the path (e.g /contacts) and url (e.g.
      localhost:300/contacts) of the current Component in the form of an object
    

    

      
      
        
import React from "react";
import Cars from "./Cars";
import Dogs from "./Dogs";
import Recipes from "./Recipes";

const Blog = () => {
    const { path, url } = useRouteMatch();
    return (
        

            
    
                
  • 
                    <Link to={`${url}/recipes`}>Recipes</Link>
                
    • 
                
  • 
                    <Link to={`${url}/dogs`}>Dogs</Link>
                
    • 
                
  • 
                    <Link to={`${url}/cars`}>Cars</Link>
                
    • 
            


            <Switch>
                <Route path={`${path}/recipes`}>
                    
                </Route>
                <Route path={`${path}/dogs`}>
                    
                </Route>
                <Route path={`${path}/cars`}>
                    
                </Route>
            </Switch>
        

    );
};
