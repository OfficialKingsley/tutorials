URLs

A URL is just a basic way of identifying a unique resource on a site. A url could mean a unique blog in a blog post

Taking amazon for example, when we go to amazon.com, we load the home page but if we click on a product, we see a relatively different link. This link is important as it always points to that product. So anytime we want to see that product, if we copy the exact link, it will always direct us to that product

in SPAs, we don't refresh the page as it clears our state. This creates a problem as we never actually update the url because we are not going to different url because we are not sending a request to the backend server to get a new html document. Each resource on a website is actually assigned a url. In SPAs we don't link to different pages so we can't even bookmark the page as it still refers to the same html page
Another disadvantage of SPA is that we can't go back and forward to the pages we visited since it is still the same domain and the same index.html that we are loading
