In react every part of the app is actually a component

React components also have state

Components have state.
These states are like the properties of the components 

state = {
  title: "This is my title",
  body: "This is the body of the book",
  isFeatured: true
}
        
import React, {Component} from "react";
class ThisComponent extends Component {
  state = {
    todos: [/*Array of todos*/]
  }
  render () {
    return (
      <Todos />
    )
  }
}

The render method is called immediately the component is initialized

The state object in class based components are actually mutable. To update a state though, we use the this.setState() function.

When you want to use a defined function or method in class components, we have to bind it to the this keyword. 

getStyle() {
  return (<Hello world>)
}
this.getStyle.bind(this)

//The above is the syntax
//This binds it to the function
//The reason that happens is because
//The this keyword is attached to the function
//The reason we can use this.state is because
//These are part of the component class which was imported from react

A way to avoid this binding is by using arrow functions

getStyle = () => {
  return (<Hello world>)
}
this.getStyle

//Now we see that we don't have to bind

However, if we want to pass in parameters into the function, we will have to bind. When we use the bind method, the first parameter has to be the this keyword and the following parameters will refer to the arguments of the function 

getStyle() {
  return (<Hello world>)
}
this.getStyle.bind(this, myFirstArgument, mySecondArgument)
