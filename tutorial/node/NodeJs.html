Why use node

It is fast, efficient and highly scalable

It is event driven, and has a non-blocking I/O model

It is popular in the industry

It uses JS

It is asynchronous and works on a single thread which holds a lot of connections from the client to server in what is called an event loop. We do not use Node in CPU intensive apps that involve so much calculation.
Since it works on a single thread asynchronously, when an event is triggered a callback is fired.

What is Node JS good for

Anything that is not CPU intensive

  • Rest APIs and Micro-servers
  • Real TIme services (Chat, Live Updates)
  • CRUD Apps - Blogs, Shopping Carts, Social Networks
    CRUD operations mean Create, Read, Update and Delete
  • Tools and Utilities
