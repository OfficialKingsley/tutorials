Installing packages with nom

A package can be defined as a folder with modules in it

npm i package_name

//in the package.json, you will see a new property called dependencies
"dependencies": {
    "package_name": "package_version"
}

//Another folder called node_modules is where all the downloaded dependencies can be found

To use the node_module all you need to do is to just require it

Semantic Versioning

This is a standard that node packages follow

In the package version e.g. if the version of loadash is
"^4.17.11"
//They represent major_version.minor_version.patch_version

If the patch version changes that means that there was a patch update and that they implemented some bug fixes.
If the minor update occurs, new functionality has been added and old functionality has been deprecated but the old functionality can still be used
If the major_version is changed, that means there was a breaking change. Tha

The caret (^) character in front of the number tells npm to install the latest versions of the package as long as its still the main version. So if there's any update in the minor version or patch version, it will be updated but not the major version
The tilder (~) This means you want only the patch update
And if none of those signs are used, there will be no update
