Serving Static Files with Http and File system modules

const http = require("http");
const fs = require("fs");
const server = http.createServer((req, res)=> {
  const readStream =fs.createReadStream("./static/index.html")
  res.writeHead(200, {
    "Content-type": "text/html"
  });
  readStream.pipe(res)

  //The res is also a writable stream
})
server.listen(3000)

//Content types
//html- text/html
//json - applicaion/json
//image/png

Semantic Versioning

This is a standard that node packages follow

In the package version e.g. if the version of loadash is
"^4.17.11"
//They represent major_version.minor_version.patch_version

If the patch version changes that means that there was a patch update and that they implementedsome bug fixes.
If the minor update occurs, new functionality has been added and old functionality has been deprecated but the old functionality can still be used
If the major_version is changed, that means there was a breaking change. Tha

The caret (^) character infront of the number tells npm to install the latest versions of the package as long as its still the main version. So if there's any update in the minor version or patch version, it will be updated but not the major version
The tilder (~) This means you want only the patch update
And if none of those signs are used, there will be no update
