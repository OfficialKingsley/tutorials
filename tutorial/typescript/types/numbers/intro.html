To prevent this, we can add the types that we want the parameters to be 

function add(n1: number, n2: number) {
  return n1 + n2
}
const number1 = '5'
const number2 = 2.8

const result = add(number1, number2)
//We get an error that we can't add string arguments to number parameters

console.log(result)

//Output: 52.8
//This is because of the concatenation

Type inference

Typescript always does its best to get the type of data stored in a variable 

let number1: number = 5; //This is bad practice

let number2: number;

number2 = "5" //Error
number2 = 5 //This then works
        
let phrase = "This is my phrase"

phrase = 0; 
//This produces an error because
//typescript inferred that the first declaration was a string
