This is one of the new types that is added to typescript but is not in javascript. It will be compiled to a javascript array
A tuple can be said to be an array with a definite length and definite types

For example for some reason you want you want an array that has only two elements 

role = [2, "author"]

//The inference shows
role: (string | number)[]

However, this is not the type that we are looking for

role.push("admin")

The above code will run even though we don't want to add items to the array (or tuple) 

let role: [number, string]

The above is therefore setting the type of the data to a tuple.
We are explicitly setting the datatypes in the array is what is called a tuple

If you type role.push("admin") this will work because push is an exception to tuples even though we have explicitly said that we want just two element in the tuple 

let role: [number, string]

role = [1, "author", "captain"]
//The above will create an error
//So you see that you can't add another item
